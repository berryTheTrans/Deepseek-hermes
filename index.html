A complete step-by-step guide to connect DeepSeek V4 Pro to Hermes Agent on Windows — the right way.

Windows 10/11
DeepSeek V4 Pro
Hermes v0.14+

// Setup Steps

01
Get Your DeepSeek API Key
Create an account on the official platform
  • Go to platform.deepseek.com and sign up / log in
  • Navigate to API Keys in the dashboard
  • Click Create new key and copy it somewhere safe
  • Make sure your account has credits — even a valid key fails with zero balance
⚠

Never share your API key publicly. Anyone with your key can use your credits. If exposed, delete it immediately and create a new one.

Available DeepSeek models:

deepseek-v4-pro
Best quality — recommended for most tasks
deepseek-v4-flash
Faster and cheaper for simple tasks
deepseek-reasoner
Deep reasoning / R1 thinking mode
deepseek-chat
Legacy alias — being deprecated Jul 2026
02
Add Your API Key to Hermes
Store it in the .env file — the correct way

Open PowerShell and run this command to open the Hermes environment file:

PowerShell 
notepad "$env:USERPROFILE\.hermes\.env"

Add this line to the file (replace with your actual key):

.env 
DEEPSEEK_API_KEY=your-api-key-here

Save and close Notepad.

03
Fix the Provider in config.yaml
The most common mistake — must be "deepseek" not "openai"

Run this single PowerShell command to automatically fix the provider setting:

PowerShell 
(Get-Content "$env:USERPROFILE\.hermes\config.yaml") `
  -replace 'provider: "openai"', 'provider: "deepseek"' `
  | Set-Content "$env:USERPROFILE\.hermes\config.yaml"

Verify the change worked:

PowerShell 
Select-String -Path "$env:USERPROFILE\.hermes\config.yaml" -Pattern "provider:"

You should see provider: "deepseek" on line ~43.

04
Set the Model and Base URL
Make sure these three lines are correct in config.yaml

Open the config file:

PowerShell 
notepad "$env:USERPROFILE\.hermes\config.yaml"

Find the model: section near the top (~line 30) and make sure it looks like this:

config.yaml 
model:
  default: "deepseek-v4-pro"
  provider: "deepseek"
  base_url: "https://api.deepseek.com"

Save and close.

05
Verify Everything Works
Run the built-in health check

Run Hermes Doctor to confirm DeepSeek is connected:

PowerShell 
hermes doctor

Under ◆ API Connectivity you should see:

✓

✓ DeepSeek — This means your API key is valid and the connection works.

06
Start Chatting
Use the CLI — most reliable on Windows

Launch Hermes chat from PowerShell:

PowerShell 
hermes chat
🎉

Type "hello" — if DeepSeek responds, you're all set! The CLI is the most reliable interface on Windows.

ℹ

The Hermes desktop GUI app is still in early beta on Windows and may not always read config changes correctly. The CLI (hermes chat) is the recommended way to use Hermes on Windows.
